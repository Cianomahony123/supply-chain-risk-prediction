# Supply Chain Risk Prediction — Multi-Domain Data Fusion

**Cian O'Mahony | Stevens Institute of Technology | CS 595**

A machine learning pipeline that predicts global shipping disruptions by fusing three heterogeneous data sources: internal logistics logs, Federal Reserve macroeconomic indices, and global weather data. The project benchmarks three architectures — Balanced SVM, MLP, and Random Forest — and identifies a critical **Accuracy-Recall Paradox** that renders high-accuracy models operationally useless in high-stakes logistics environments.

---

## The Core Problem

Traditional ERP systems treat supply chain risk as a single-domain problem. This project builds a **Tri-Factor fusion model** that captures both:

- **Systemic risk** — New York Fed Global Supply Chain Pressure Index (GSCPI): macro-level friction from inflation, labor shortages, and geopolitical shifts
- **Stochastic risk** — localized meteorological data: wind gusts and precipitation at destination ports
- **Operational features** — shipment distance, cargo weight, and lead time from internal logistics logs

## Results

| Model | Accuracy | Recall (Disruptions) | Precision | Latency |
|---|---|---|---|---|
| **Balanced SVM (RBF)** | 61.40% | **0.74** | 0.33 | **0.276s** |
| MLP Neural Network | 72.60% | 0.20 | 0.31 | 3.295s |
| Random Forest | 71.10% | 0.18 | — | 0.450s |

**The Balanced SVM is the operationally superior model.** The MLP's 72.6% accuracy is a "False Superiority" — it achieved high nominal accuracy by defaulting to the majority class (on-time shipments) and correctly identified only 20% of actual disruptions. In logistics, a missed disruption (False Negative) triggers liquidated damages, missed contract penalties, and lost customer trust. A false alarm (False Positive) costs a precautionary reroute — expensive but recoverable.

## Key Engineering Decisions

### 1. Temporal Normalization
Logistics logs are daily; the GSCPI is monthly. Rather than interpolating economic data, a shared `YearMonth` key was used to join them — grounding every shipment in its true macroeconomic context.

### 2. Macro-Regional Proxy Mapping
40% of destination ports had no direct weather station data. Instead of deleting those rows or filling with zeros (which would falsely imply perfect weather), each port was mapped to its host country's national climate averages. This preserved the volatility signal across 100% of records.

```python
port_to_country = {
    'Los Angeles': 'United States of America',
    'Shanghai': 'China',
    'Rotterdam': 'Netherlands',
    'Busan': 'South Korea',
    'Dubai': 'United Arab Emirates'
}
```

### 3. Z-Score Standardization
Shipping distances run to 14,000+ km while GSCPI values hover between -1 and 4. Without scaling, distance would numerically dominate the SVM's margin calculation and drown out the economic signal entirely.

### 4. Cost-Sensitive Objective Function (The Breakthrough)
Early SVM versions achieved 78% accuracy with **0.00 recall** for disruptions — the model had learned to guess "on-time" every time. Implementing `class_weight='balanced'` assigned a ~3.5× heavier penalty to missed disruptions, forcing the model to accept a narrower margin in exchange for catching them. Recall jumped from 0.00 to **0.74**.

```python
svm_model = SVC(kernel='rbf', class_weight='balanced', random_state=42)
```

### 5. RBF Kernel for Non-Linear Risk Boundaries
Supply chain risk is conditional: high wind only causes a delay if the port is already economically congested. A linear classifier cannot separate these compounding "Islands of Risk." The Radial Basis Function kernel projects the feature space into a higher-dimensional space where these compound risk zones become separable.
